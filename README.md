יומן המשימות שלי (React Task Manager)

אפליקציית מנהל משימות שנבנתה ב־React + Vite במסגרת פרויקט מסכם.
האפליקציה תומכת בניהול משימות מקצה לקצה: הוספה, עריכה, סימון כהושלם, מחיקה (soft-delete), סינון לפי סטטוס ושמירה מתמשכת ב־LocalStorage.

מטרות ויכולות עיקריות

הוספת משימה חדשה הכוללת:

תיאור משימה (text)

תאריך יעד (dueDate)

אחראי (assignee)

סטטוס (חדש / בתהליך / תקוע / הושלם)

עריכת משימות קיימות (כולל שינוי תאריך יעד/אחראי/סטטוס)

סימון משימה כהושלמה/לא הושלמה

מחיקה רכה (Soft Delete): משימות שנמחקות אינן מוצגות אך נשמרות במערכת (LocalStorage)

סינון תצוגה: הכל / פעיל / הושלם + הדגשת סינון נבחר

הצגת סטטיסטיקות: מונה משימות פעילות וסיכום סטטוסים

שמירה וטעינה אוטומטית ל־LocalStorage תחת המפתח: tasks_v1

נרמול נתונים בטעינה (migration) עבור משימות מגרסאות קודמות שחסרים להן שדות

פירוט קומפוננטות ואחריות

App.jsx
רכיב השורש. אחראי על ניהול ה־state הראשי של המשימות, טעינה/שמירה ל־LocalStorage, נרמול נתונים בעת טעינה, לוגיקת סינון, והעברת handlers ו־props לרכיבי המשנה.

components/TaskInput.jsx
טופס הוספת משימה: קליטת טקסט, תאריך יעד, אחראי וסטטוס. מבצע ולידציה ויוצר משימה חדשה.

components/TaskList.jsx
מציג את רשימת המשימות לאחר סינון. יוצר רכיבי TaskItem עבור כל משימה.

components/TaskItem.jsx
מציג משימה בודדת, כולל:

toggle השלמה

עריכה inline של תוכן/דדליין/אחראי/סטטוס

הצגת מידע נוסף (למשל תאריך יצירה וזמן מאז יצירה)

components/FilterBar.jsx
שליטה בסינון תצוגה (הכל / פעיל / הושלם) והדגשת הפילטר הנבחר.

components/FooterStats.jsx
סטטיסטיקות תחתונות: מונה משימות פעילות, סיכום לפי סטטוסים, וכפתור “נקה משימות שהושלמו” (Soft Delete למשימות שהושלמו).

מודל הנתונים

כל משימה נשמרת כאובייקט:

{
  "id": "<uuid>",
  "text": "תיאור המשימה",
  "completed": false,
  "deleted": false,
  "createdAt": "2026-02-03T00:00:00.000Z",
  "dueDate": "YYYY-MM-DD",
  "assignee": "שם אחראי",
  "status": "חדש|בתהליך|תקוע|הושלם",
  "completedAt": "2026-02-03T00:00:00.000Z"
}


בעת טעינת האפליקציה מתבצע נרמול נתונים כדי לתמוך במשימות מגרסאות קודמות שחסרים להן שדות (למשל status, deleted, createdAt וכדומה).

שמירה ושחזור נתונים

הנתונים נשמרים תחת tasks_v1 ב־LocalStorage כמערך JSON.

משימה “נמחקת” מוגדרת כ־deleted: true ולכן לא תוצג בממשק, אך תשמר ב־LocalStorage (Soft Delete).

בדיקות

כרגע אין בדיקות יחידה בפרויקט.
ניתן להרחיב ולהוסיף Vitest + React Testing Library בהמשך.

מגבלות/באגים ידועים

שיפור נדרש בנגישות מקלדת לתפריט בחירת סטטוס (ניווט עם חצים וסגירה עם Escape).

אין בדיקות יחידה כברירת מחדל.

מחיקה היא Soft Delete בלבד — למחיקה פיזית יש לנקות LocalStorage או להוסיף מנגנון purge.

דגשים להגשה וביקורת

בעת הצגה/הגשה מומלץ לדעת להסביר:

היכן מנוהל ה־state הראשי ולמה (App.jsx)

זרימת מידע: רכיבי משנה (children) מפעילים handlers שמוגדרים ב־Parent

כיצד מתבצעת שמירה ל־LocalStorage (Persistence) ומדוע נדרש נרמול בעת טעינה

לוגיקת השלמה וסטטוס:

סטטוס “הושלם” מסמן completed: true וממלא completedAt

React + Vite (תשתית הפרויקט)

הפרויקט מבוסס על תבנית React עם Vite הכוללת HMR וכללי ESLint בסיסיים.
ההרצה מתבצעת באמצעות npm run dev וה־build יתבצע באמצעות npm run build (אם נדרש).
